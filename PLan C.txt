Core Shape

Use a hybrid controller: sparse proofgraph/global routing chooses where to go next, and a local motion controller chooses how to move for the next few ticks. It should plug into the existing BotBrain path where BotBrainController.cs (line 167) currently calls proof routes, graph steering, and PrimitiveDirectDrive.cs (line 47).

The key rule: no runtime tree search. The controller evaluates a tiny fixed set of macro actions only on replan ticks, commits to one for several ticks, and otherwise just replays the chosen steering.

Decision Ladder

Proven cached primitive for same map/class/team/carrying/surface/target bucket.
Cheap same-surface direct drive.
Bounded local macro probe.
Global lane/proofgraph attach target.
Idle/defend/follow-carrier fallback.

Same-surface direct drive is allowed when current surface equals target surface, target vertical delta is small, no gate/wall blocks the run, and target movement stayed inside the same bucket. Otherwise use a probe or attach target.

Core Algorithm

csharp



bool TryResolve(ctx, out steering, out trace) {
    goal = ResolveGoal(ctx); // dropped intel, carrier, proof route entry, lane portal, etc.

    if (state.ActivePlan is valid && !ShouldAbort(ctx, goal)) {
        steering = ExecuteActivePlan(ctx);
        return true;
    }

    if (IsSameSurfaceSimpleDrive(ctx.Self, goal, ctx.World, out steering)) {
        state.ActivePlan = null;
        trace = "local=sameSurface";
        return true;
    }

    if (!IsPlanTick(ctx.Tick) && !GoalBucketChanged(goal)) {
        steering = ctx.BaseSteering;
        return false;
    }

    key = CacheKey.From(ctx, goal);
    if (cache.TryGetPositive(key, out plan) && !suppressor.IsSuppressed(plan)) {
        state.ActivePlan = plan.WithCommit(ctx.Tick);
        steering = ExecuteActivePlan(ctx);
        trace = "local=cacheHit";
        return true;
    }

    using budget = ctx.Budget.TryAcquireBotSlice(maxMs: 0.25);
    if (!budget.Available) {
        steering = ResolveAttachOrBase(ctx, goal);
        return true;
    }

    candidates = BuildCandidates(ctx, goal).Take(8);
    best = EvaluateCandidates(ctx, goal, candidates, maxSimTicks: 96, budget);

    if (best.Score >= MinAcceptScore && best.Improvement >= MinImprovement) {
        cache.StorePositive(key, best.Plan);
        state.ActivePlan = best.Plan.WithCommit(ctx.Tick);
        steering = ExecuteActivePlan(ctx);
        trace = best.Trace;
        return true;
    }

    cache.StoreNegative(key);
    steering = ResolveAttachOrBase(ctx, goal);
    trace = "local=noCandidate";
    return true;
}



Candidate Set

Keep this deliberately small:

text



RunLeft/RunRight: 12, 18, 24 ticks
Brake: 8 ticks
GroundJumpLeft/GroundJumpRight: jump on tick 0, run 18 or 24 ticks
ObstacleHopLeft/Right: run into wall, jump once, continue 24 ticks
DropLeft/DropRight: down + direction for 8-14 ticks
Fall/Wait: 8-12 ticks when airborne or unstable
AirJumpLeft/Right: only if RemainingAirJumps > 0 and target/anchor is above
AttachEntryLeft/Right: same as run/jump, but scored against nearest proof/lane portal



Do not add class-special abilities in stage one. Scout air jump can be supported because it is already normal movement state. Soldier rocket jumps, sticky jumps, and weapon movement should be later opt-in primitives with separate metrics.

Simulation

Use movement-only cloned players and the same PlayerEntity.Advance(...) path used by offline probing in BotBrainMovementProbe.cs (line 507). The clone must never mutate SimulationWorld.

csharp



SimResult Simulate(ctx, candidate, goal) {
    player = MovementSimPlayerFactory.CloneMovementState(ctx.Self);

    for tick in 0..candidate.DurationTicks {
        input = candidate.BuildInput(tick, player, goal);
        jumpPressed = input.Up && !prev.Up;
        player.Advance(input, jumpPressed, ctx.World.Level, ctx.Team, FixedDt);

        if (!player.IsAlive || IsOutOfBounds(player)) return Fatal;
        if (ReachedGoalOrAttachSurface(player, goal)) break;
    }

    return Measure(player, ctx.Self, goal);
}



Scoring

Use surface/anchor progress, not raw distance alone:

csharp



score =
    + 4.0 * GoalDistanceReduction
    + 3.0 * SurfaceProgress               // target surface, route surface, lane portal
    + 2.0 * StableGroundedLandingBonus
    + 2.0 * ProofRouteAttachBonus
    + 5.0 * PickupOrCaptureRadiusBonus
    + 0.5 * ForwardVelocityTowardAnchor
    - 8.0 * DeathOrInvalidPenalty
    - 3.0 * WrongVerticalLayerPenalty
    - 2.0 * AirborneUnstablePenalty
    - 4.0 * SuppressedPrimitivePenalty
    - 2.0 * DirectionFlipPenalty
    - 3.0 * RepeatedJumpPenalty;



For CTF lanes, target anchors should be lane-aware: lower, middle, upper, connector. A dropped intel below the bot should prefer a lower-lane/drop anchor; an upper-platform target should prefer an upper lane or proofgraph entry. This prevents “nearest Euclidean point” from choosing the wrong layer.

Budget Model

At 30 TPS:

text



Normal tick per bot: O(1), replay active macro.
Replan cadence: every 8-15 ticks, staggered by slot.
Candidates per replan: max 8.
Horizon per candidate: max 12-32 ticks.
Total simulated ticks per replan: max 96.
Stopwatch budget: 0.20-0.25 ms per bot replan.
Global frame budget: 1.0-1.5 ms for all bot local planning.



If the stopwatch expires, stop evaluating candidates and choose the best valid candidate seen so far, or fall back. Complexity is hard-capped: O(botCount) per live tick with bounded occasional candidateCount * horizon.

Cache value stores the macro, confidence, expected end surface, EWMA success rate, last failure tick, and expiry. Positive cache entries are reused immediately. Negative entries suppress failed jumps/drops for a short TTL, then decay.

Proofgraph Integration

Proofgraphs remain high-confidence lanes. The local controller should:


Target dropped intel directly only when it is locally plausible.

Otherwise target nearest proof route entry, lane portal, or verified surface.

Hand control back to VerifiedNavProofRouteExecutor once inside attachment tolerance.

Treat old generated nav graph edges as low-confidence anchors only, not proof of traversal.

When proof execution aborts due to divergence/start-outside-tolerance, ask local motion for an attach macro instead of waiting at the intel room.


This should enhance the current proof route executor in VerifiedNavProofRouteExecutor.cs (line 257), not replace it.

Anti-Churn Rules


Commit to selected macro for at least 6-12 ticks unless death, large target bucket change, or severe divergence.

Require minimum score improvement before switching plans.

Suppress a failed primitive by (surface, target bucket, class, direction, primitive) for 2-10 seconds.

Track best distance/score; abort after stagnant window, then blacklist that macro.

Add jump cooldown and repeated-jump penalty.

Do not replan because a carrier moved a few pixels; bucket moving targets.

Prefer stable grounded outcomes over slightly closer airborne outcomes.

If no candidate improves, choose attach-to-lane or defend/follow fallback, not another blind jump.
