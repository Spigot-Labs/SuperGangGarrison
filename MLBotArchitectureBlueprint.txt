MLBot Architecture Blueprint: Superseded By Failure Handoff

The prior blueprint is intentionally removed from this document because the implemented direction did not produce reliable bots.

The useful architecture lesson is negative:

- A model trained around brittle route/waypoint scaffolding did not become a reliable general traversal policy.
- A headless improvement loop was not proven.
- A local movement oracle was not made reliable enough to act as a teacher.
- TwodFortTwo remained unresolved for broad full-score coverage.

See `handoff.txt` and `MLBot_STATUS.md` for the failure record and the concrete dead ends.

Any future blueprint should start from a proven deterministic traversal/score baseline, then decide whether ML has a role. It should not assume the current MLBot stack is release-ready.
